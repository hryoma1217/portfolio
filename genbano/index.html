Problem & Solution

「現場」と「事務所」の分断を、設計で埋める

中小建設会社の事務は、転記・二重入力・属人化が常態化していました。それをひとつの業務フローに統合します。

課題 Before

  • 日報は紙、見積はExcel、台帳はAccess。データが分断し転記ミスが頻発
  • 現場の出来高が事務所に伝わらず、請求・原価の把握が遅れる
  • 高齢の職人が多く、複雑な業務アプリは現場で使われない
  • 給与計算が手作業で、手当・車代・按分の計算が属人化

解決 With Genbano

  • 日報→工事→見積→受注→請求→入金→給与を一本のデータでつなぐ
  • 現場はLINE/LIFFから日報を入力するだけで原価・請求へ自動反映
  • 低ITリテラシー層でも迷わない、徹底的に簡素化したUI
  • 日給月給・時給按分の2モードで給与を自動計算
Features

建設業の実務に合わせた機能群

汎用的な業務ツールではなく、建設業の請求・原価・現場運用に特化した機能を備えています。

モバイル日報・勤怠

現場の職人がLINE/LIFFからその場で日報と勤怠を入力。事務所への報告の手間と転記をゼロに。

工事・案件管理

案件ごとに日報・原価・見積・請求を集約。プロジェクト単位で進捗と収支を一望できます。

見積・受注・請求

見積から受注、請求までを連続処理。出来高請求（契約額×累計出来高率−前回請求）や分割請求にも対応。

給与計算 2モード

日給月給（simple）と時給按分（detailed）を切替。手当・車代・調整金を含め日報から自動集計。

帳票出力 (Excel/PDF)

アップロードした自社Excelテンプレートに見積書・請求書データを差し込み、Excel/PDFを自動生成。

添付ファイル管理

写真・図面・契約書をロール別の公開範囲で保管。現場には必要な情報だけを安全に共有。

End-to-End Workflow

日報から給与まで、ひとつのデータで流れる

各ステップが分断せず連動。現場の入力が、最終的に給与計算と収支把握まで自動でつながります。

Engineering Highlights

設計で効いたポイント

建設業SaaSとして特に作り込みが必要だった3つの観点です。

SECURITY

マルチテナント × ロール権限

管理者 / 事務 / 現場の3ロールできめ細かく制御。現場ロールは金額を見られない （canViewMoney）、案件ごとの参加者制御（project_members）、 操作の最小権限（min_role）など、建設現場の情報統制を反映。テナント=DB分離で境界を明確化。

COST

ランニングコスト ¥0

無料枠で完結するCloudflare D1 + R2 + Workers構成へ移植。中小企業が月額の固定費を気にせず 導入できることを最優先に、インフラ選定からアーキテクチャを再設計しました。

UX

低ITリテラシーUX

高齢の職人でも迷わないことを設計上の最重要KPIに設定。入力はLINE/LIFFに寄せ、 項目数を絞り、専門用語を避けた導線で「現場で実際に使われる」状態を作りました。

これはプライバシー配慮のため匿名化したケーススタディです。実在のクライアント名・企業名・個人名・ドメイン・リポジトリは一切含みません。 「地方の建設会社向けに開発した業務システム（建設業向けSaaS）」として、技術構成とアーキテクチャのみを紹介しています。
