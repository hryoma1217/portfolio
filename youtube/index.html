お題を渡すだけで、台本生成・音声合成・立ち絵の口パク・字幕・背景・サムネ・YouTube 投稿までを一気通貫で自動化した動画生成パイプライン。 長尺（ゆっくり / ずんだもん）と Shorts の両系統を、同じ仕組みで継続運用しています。

人手の介入は「お題の指定」と「公開前チェック」だけ。生成・合成・下書き投稿までは無人で完走します。

01できること

お題だけで1本まるごと生成

テーマを渡すと、台本・音声・立ち絵動画・字幕・サムネ・概要欄まで自動生成。最後の公開判断だけ人が行います。

2キャラの掛け合い台本

解説役と聞き役の会話形式で構成。LLM で下書きし、誤読・口調を機械チェックしてから音声化します。

立ち絵の口パク・まばたき

音量解析（RMS）で発話区間を検出し、話している側だけ口を開閉。表情・まばたきも自動で動かします。

長尺と Shorts を同じ仕組みで

10〜15分の長尺と縦型 Shorts を共通の中間形式から出力。スキンを切り替えるだけで声・立ち絵・演出が変わります。

02処理の流れ

1. 台本生成

LLM もしくは事前生成（prefetch）で会話台本を作成。話者・章立て・サムネ文言・読み辞書を構造化 JSON にまとめます。

2. 音声合成

ずんだもんは VOICEVOX、ゆっくりは AquesTalk で合成。固有名詞はユーザー辞書で誤読を事前補正します。

3. 映像合成

口パク・字幕・章タイトル・背景・BGM / SE を FFmpeg のフィルタグラフで一括合成し、1本の mp4 に仕上げます。

4. メタデータと投稿

タイトル・概要・章タイムスタンプ・タグを自動生成し、YouTube Data API で下書き（非公開）投稿まで実行します。

032つの動画系統

長尺 — ゆっくり / ずんだもん

10〜15分の解説動画。章立て・ミッドロール設計・サムネ最適化まで含めて1本を組み上げます。

Shorts — 縦型・短尺

テーマ別に毎日量産する縦型ショート。長尺への相互誘導リンクも自動で差し込みます。

04技術的なポイント

共通中間形式で多出力

話者を抽象化した台本 JSON を起点に、スキン（声・立ち絵・色・演出）を差し替えて複数パターンを同一コードで出力。

誤読を機械チェック

合成前に全行の読み仮名を検証し、辞書登録や表記修正で固有名詞のミスを潰してからレンダーします。

長尺でも破綻しない合成

口パクの区間結合やフィルタのスクリプト化で、長尺レンダー時のメモリ枯渇・容量爆発を回避。

相互誘導まで自動化

投稿後に再生リスト追加・Shorts ⇔ 長尺の概要欄リンク・誘導コメントを自動付与し、回遊を設計します。
